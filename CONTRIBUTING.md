# Contributing

Thank you for your interest in contributing!

## Getting Started

1. Fork the repository and clone it locally
2. Create a feature branch: `git checkout -b feature/my-feature`
3. Make your changes and add tests where appropriate
4. Run the test suite to make sure everything passes
5. Commit your changes with a clear message
6. Open a Pull Request

## Code Style

- Follow the existing code style in the project
- Run the linter/formatter before submitting (`pre-commit run --all-files`)
- Keep pull requests focused — one feature or fix per PR

## Reporting Issues

Please use GitHub Issues and include:
- A clear description of the problem
- Steps to reproduce
- Expected vs actual behaviour
- Environment details (OS, language version, etc.)

## Pull Request Guidelines

- Reference any related issues in the PR description
- Keep the PR description concise but informative
- Be responsive to review feedback
