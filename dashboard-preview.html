User Guide

Everything you need to know about SevaForge

Getting Started

SevaForge generates production-ready infrastructure for any codebase. Just point it at a folder, hit Run, and review the output.

1. Select a project folder using Browse or paste a path
2. Click "Run Full Pipeline" to start analysis
3. Monitor progress across 16 stages in real-time
4. Review generated files and push to GitHub

Pipeline Phases

Analyse

Scans your codebase, detects the tech stack, normalizes project structure, generates documentation, and captures your intent.

Build

Generates Terraform, Dockerfiles, ArgoCD configs, GitHub Actions workflows, and architecture designs tailored to your stack.

Quality

Creates Playwright E2E tests, runs AI-powered code reviews, performs security scans for CVEs and secrets, and cross-validates all outputs.

Ship

Sets up Prometheus + Grafana observability, assembles the final package, runs quality gates, and delivers everything to GitHub.

AI Agents

Seven specialized agents collaborate on every pipeline run. Each focuses on a domain and validates its work against the others.

Scout Discovers project structure, dependencies, and tech stack
Architect Designs infrastructure topology and deployment strategy
Builder Generates IaC, CI/CD, and container configurations
Tester Creates E2E test suites from your app's routes
Guardian Scans for vulnerabilities, secrets, and compliance issues
Scribe Writes README, API docs, runbooks, and architecture docs
Orchestrator Coordinates all agents and resolves conflicts

Intelligence Maturity

SevaForge learns from your decisions over time. Each stage progresses through 4 maturity phases:

Phase 1 Assisted — suggests defaults, you confirm every time
Phase 2 Automated — applies stable patterns, flags changes only
Phase 3 Augmented — adapts to new patterns, learns from feedback
Phase 4 Agentic — fully autonomous decisions with high trust scores

Tips

• Use Dry Run to preview outputs without writing any files
• Check the Intel tab to see your trust scores evolve over time
• The History tab shows all previous pipeline runs and their results
• Hover over the ? icons for context on any section
• CLI command: forgeflow dashboard to launch this UI
